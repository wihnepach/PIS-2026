## Use-case: Запуск Pomodoro‑сессии

**Первичный актор**: Пользователь

**Цель**: Начать Pomodoro‑сессию для выбранной задачи и получить уведомление о её завершении.

**Предусловия**: 
- Пользователь авторизован в системе.
- В системе существует хотя бы одна активная задача.
- Нет другой активной Pomodoro‑сессии.

**Основной поток** (Happy Path):
1. Пользователь открывает список задач.
2. Пользователь выбирает задачу.
3. Пользователь нажимает «Начать Pomodoro».
4. Система создаёт новую Pomodoro‑сессию и фиксирует время старта.
5. Система запускает таймер (25 минут).
6. По истечении времени система завершает сессию и фиксирует длительность.
7. Система обновляет статистику пользователя.
8. Система отправляет уведомление о завершении.
9. Use-case завершается успешно.

**Постусловия**:
- Создана и сохранена новая Pomodoro‑сессия.
- Статистика пользователя обновлена.
- Пользователь получил уведомление.

**Альтернативные потоки**:
- 3a. Если пользователь выбрал завершённую задачу:
  - 3a1. Система отображает сообщение об ошибке.
  - 3a2. Возврат к шагу 2.

- 6a. Если пользователь завершает сессию вручную:
  - 6a1. Пользователь нажимает «Завершить».
  - 6a2. Система фиксирует фактическую длительность.
  - 6a3. Возврат к шагу 7.

- 6b. Если пользователь ставит сессию на паузу:
  - 6b1. Пользователь нажимает «Пауза».
  - 6b2. Система фиксирует время паузы.
  - 6b3. Пользователь нажимает «Продолжить».
  - 6b4. Система возобновляет таймер.
  - 6b5. Возврат к шагу 5.

**Исключительные ситуации**:
- 3b. Если произошла ошибка при создании сессии:
  - 3b1. Система откатывает транзакцию.
  - 3b2. Система уведомляет пользователя.
  - 3b3. Use-case завершается неудачей.

- 5a. Если таймер не может быть запущен:
  - 5a1. Система помечает сессию как Failed.
  - 5a2. Система уведомляет пользователя.
  - 5a3. Use-case завершается неудачей.
