## Ход выполнения работы

### 1. Session Service

**Bounded Context:** Управление Pomodoro-сессиями

**API:**
- POST `/api/sessions`
- POST `/api/sessions/{id}/finish`
- POST `/api/sessions/{id}/fail`

---

### 2. Notification Service

**Bounded Context:** Управление уведомлениями

**API:**
- POST `/api/notifications`

---

### 3. Get Service

**Bounded Context:** Сервис получения данных

**API:**
- GET `/api/get/sessions`
- GET `/api/get/sessions/{id}`

---

### 4. Event Bus (RabbitMQ)

**События:**
- `SessionStarted`
- `SessionCompleted`
- `SessionFailed`

**Скриншот RabbitMQ Management:**

![E2E-tests](./rabbit.jpg)

---

### 5. API Gateway

**Маршрутизация:**
- `/api/sessions/**` → Session Service
- `/api/notifications/**` → Notification Service
- `/api/get/**` → Get Service

**Конфигурация:**
```nginx
events {}

http {
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    upstream session_service {
        server session-service:8000;
    }

    upstream notification_service {
        server notification-service:8001;
    }

    upstream get_service {
        server get-service:8002;
    }

    server {
        listen 80;

        # Session Service
        location /api/sessions/ {
            proxy_pass http://session_service;
        }

        # Notification Service
        location /api/notifications/ {
            proxy_pass http://notification_service;
        }

        # Get Service
        location /api/get/ {
            proxy_pass http://get_service;
        }
    }
}
```

## Вывод

В ходе выполнения лабораторной работы была проведена декомпозиция приложения на независимые микросервисы, каждый из которых отвечает за свой bounded context. Были выделены три сервиса: Session Service, Notification Service и Get Service, что позволило разделить ответственность и упростить дальнейшее развитие системы.

Для обеспечения асинхронного взаимодействия между сервисами был интегрирован RabbitMQ в качестве Event Bus.  
События `SessionStarted`, `SessionCompleted` и `SessionFailed` используются для передачи изменений состояния сессии между сервисами.

Также был реализован API Gateway, обеспечивающий единую точку входа и маршрутизацию запросов к нужным сервисам.
