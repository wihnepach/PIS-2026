## Ход выполнения работы

### 1. Протофайлы (.proto)

**session_service.proto:**
```protobuf
syntax = "proto3";

package session;

service SessionService {
  rpc CreateSession (CreateSessionRequest) returns (SessionResponse);
  rpc GetSession (GetSessionRequest) returns (SessionResponse);
  rpc StreamActiveSessions (Empty) returns (stream SessionResponse);
}

message Empty {}

message CreateSessionRequest {
  string session_id = 1;
  string task_id = 2;
  string user_id = 3;
  int32 duration_minutes = 4;
}

message GetSessionRequest {
  string session_id = 1;
}

message SessionResponse {
  string session_id = 1;
  string task_id = 2;
  string user_id = 3;
  int32 duration_minutes = 4;
  string status = 5;
}
```

### 4. Server-Side Streaming

**Сценарий:**
Клиент подписывается на поток активных сессий, после чего сервер периодически отправляет данные обо всех активных сессиях.

## Контрольные вопросы

1. **Что такое gRPC?**  
   gRPC — это высокопроизводительный фреймворк удалённого вызова процедур (RPC), использующий Protocol Buffers для сериализации данных.

2. **Чем gRPC лучше REST?**  
   - быстрее за счёт бинарного формата
   - строгая типизация через .proto
   - поддержка streaming

3. **Что такое Protocol Buffers?**  
   Это язык описания структуры данных и механизм сериализации, используемый в gRPC.

4. **Что такое streaming в gRPC?**  
   Это механизм передачи потока данных между клиентом и сервером (например, server-side streaming).
