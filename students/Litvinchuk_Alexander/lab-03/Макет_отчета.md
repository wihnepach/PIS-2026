```python
# /domain/models/duration.py

from src.domain.exceptions.domain_exceptions import InvalidSessionStateException


class Duration:
    """Value Object: длительность сессии"""

    def __init__(self, minutes: int):
        if minutes <= 0:
            raise InvalidSessionStateException("Длительность должна быть больше 0")

        self._minutes = minutes

    @property
    def minutes(self):
        return self._minutes

    def __eq__(self, other):
        return isinstance(other, Duration) and self._minutes == other._minutes

    def __hash__(self):
        return hash(self._minutes)
```

# 2. Entities (Сущности)

## Созданные Entity и Value Objects

## 1. Session – Сессия Pomodoro для задачи пользователя
- **ID поле:** `session_id`  
- **Бизнес-правила:**
  - Сессия создаётся в статусе `ACTIVE`  
  - Завершить (`finish`) можно только активную сессию  
  - При завершении сессии создаётся событие `SessionCompleted`  
  - При ошибке создаётся событие `SessionFailed`  
- **Файл:** `examples/domain/models/session.py`  

```python
from datetime import datetime
from domain.models.session_status import SessionStatus
from domain.events.session_events import SessionStarted, SessionCompleted, SessionFailed
from domain.exceptions.domain_exceptions import InvalidSessionStateException

class Session:
    def __init__(self, session_id: str, task_id: str, user_id: str, duration):
        self.session_id = session_id
        self.task_id = task_id
        self.user_id = user_id
        self.duration = duration  # Ожидается объект Duration
        self.status = SessionStatus.ACTIVE
        self.started_at = datetime.now()
        self.events = []

        # Доменное событие: сессия начата
        self.events.append(
            SessionStarted(
                session_id=session_id,
                task_id=task_id,
                user_id=user_id,
                occurred_at=datetime.now()
            )
        )

    def finish(self):
        if self.status != SessionStatus.ACTIVE:
            raise InvalidSessionStateException("Можно завершить только ACTIVE сессию")

        self.status = SessionStatus.DONE

        # Доменное событие: сессия завершена
        self.events.append(
            SessionCompleted(
                session_id=self.session_id,
                duration_minutes=self.duration.minutes,
                occurred_at=datetime.now()
            )
        )

    def fail(self, reason: str = "Unknown error"):
        self.status = SessionStatus.FAILED

        # Доменное событие: сессия завершилась с ошибкой
        self.events.append(
            SessionFailed(
                session_id=self.session_id,
                reason=reason,
                occurred_at=datetime.now()
            )
        )
```

### 3. Aggregate Root (Корневой агрегат)

**Aggregate Root:** Session

**Границы агрегата:**
- Корень: `Session`
- Внутренние сущности: нет (агрегат состоит только из корневой сущности)
- Value Objects: `Duration`

**Инварианты агрегата:**

| № | Инвариант | Как проверяется |
|---|----------|----------------|
| 1 | Сессия создаётся только со статусом ACTIVE | В конструкторе `__init__()` задаётся `self._status = SessionStatus.ACTIVE` |
| 2 | Завершить можно только активную сессию | В методе `finish()` проверяется `if self._status != SessionStatus.ACTIVE` и выбрасывается `InvalidSessionStateException` |
| 3 | При завершении сессии создаётся событие `SessionCompleted` | В методе `finish()` событие `SessionCompleted` добавляется в `self._events` |
| 4 | При ошибке сессии создаётся событие `SessionFailed` | В методе `fail()` событие `SessionFailed` добавляется в `self._events` |
| 5 | Длительность сессии должна быть положительным числом | В классе `Duration` проверка `if minutes <= 0` и выбрасывается `InvalidSessionStateException` |

## Контрольные вопросы

1. **В чём отличие Value Object от Entity?**  
   Value Object определяется исключительно значением и не имеет собственного идентификатора.  
   Entity имеет уникальный идентификатор и сохраняет свою индивидуальность даже при изменении полей.  
   В моей системе пример Value Object — `Duration`, а пример Entity — `Session`.

2. **Почему Aggregate Root должен инкапсулировать доступ к внутренним сущностям?**  
   Это гарантирует соблюдение инвариантов и предотвращает изменение состояния агрегата в обход доменной логики.  
   Только Aggregate Root контролирует корректность операций и последовательность изменений.

3. **Какая роль Domain Events? Приведите пример из вашей системы.**  
   Domain Events фиксируют значимые изменения в предметной области и позволяют другим частям системы реагировать на них асинхронно.  
   В моей системе пример — событие `SessionCompleted`, которое создаётся при успешном завершении Pomodoro‑сессии.

4. **Как вы проверяете инварианты в вашем агрегате? Приведите пример.**  
   Инварианты проверяются внутри методов агрегата и конструкторов Value Objects.  
   Например, в `Session.finish()` проверяется, что статус равен `ACTIVE`, иначе выбрасывается `InvalidSessionStateException`.

5. **Почему Value Objects делаются иммутабельными?**  
   Иммутабельность гарантирует предсказуемость поведения, отсутствие скрытых побочных эффектов и упрощает проверку инвариантов.  
   Если значение нужно изменить — создаётся новый объект, что делает модель более надёжной.

## Вывод

Реализован изолированный доменный слой Pomodoro‑системы, включающий агрегат `Session`, сущности `Task` и `User`, а также Value Object `Duration`. Все ключевые инварианты проверяются внутри конструктора и методов агрегата, что гарантирует корректность переходов состояния. Доменный слой полностью отделён от технических деталей: он не зависит от инфраструктуры, внешних сервисов или хранилищ, что обеспечивает его тестируемость и устойчивость к изменениям.
